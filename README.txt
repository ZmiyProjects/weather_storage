Для успешной установки и работы приложения необходимо следующее ПО:
–	Microsoft SQL Server 2019 для linux, для установки необходимо обладать правами администратора SA
–	mssql-tools включающий средства командной строки (sqlcmd) для linux, используемые при создании баз данных
–	Python 3.6.x-3.7.x, версия 3.8 не поддерживается так как драйвер взаимодействия с MS SQL pyodbc на текущий момент не поддерживает python 3.8

Для установки ПО необходимо скачать копию и запустить скрипт install.sh. Он принимает 3 аргумента:
–	Путь к директорию, в которую будет произведена установка (обязательный)
–	Количество обработчиков – виртуальных вычислительных потоков веб-сервиса, обеспечивающих параллельные обработку клиентских запросов, по умолчанию установлен рекомендованный параметр – число ядер процессора * 2 + 1, изменение не рекомендуется (необязательный параметр)
–	ip и порт, на которых будет запускаться приложение, по умолчанию 0.0.0.0:8080 (необязательный аргумент)

Если заданы корректные аргументы начнется установка, в процессе сначала будет запрошен пароль администратора (SA) MS SQL, затем пароль администратора linux. В результате копия ПО будет установлена в указанную директорию.
По умолчанию веб-сервис находится в неактивном состоянии, для его запуска необходимо отправить unit-файл в системный каталог linux /etc/systemd/. Все процессы связанные с запуском, приостановкой и удалением веб-сервиса автоматизированы и предоставляются посредством скрипта commander.sh, который принимает аргумент- команду из следующего перечня, Таблица Б-1. Если команда не указана или переданы недопустимая команда, 
Таблица Б-1 Перечень допустимых команд для веб-сервиса
Команда	Описание
start	Запускает работу веб сервиса
stop	Останавливает работу веб сервиса
status	Выводит в консоль статус работы веб приложения включая все обработчики
uninstall	Удаление установленной копии веб-сервиса, при этом директория установки не удаляется. Также не осуществляется удаление базы данных
